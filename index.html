4. Usability Testing

Testing Goals

  • Evaluate whether the prototype is easy to understand and navigate
  • Assess accessibility for different profiles (student, elderly user, caregiver)
  • Identify usability issues in navigation and interaction
  • Evaluate emotional response (stress, confidence, reassurance)

Participants & Setup

  • 3–5 participants
  • Moderated, in-person or remote (screen share)
  • Tool: clickable prototype (Figma)
  • Duration: 20–30 minutes each

User Tasks

  1. Add a new medication and set a daily reminder
  2. Respond to a medication reminder (confirm or snooze)
  3. Check the daily medication schedule
  4. View medication history
  5. (Caregiver only) Check whether a medication was taken

How feedback was analyzed

Feedback was grouped into recurring issues and prioritized by frequency and severity, then used to iterate on navigation clarity, labels, hierarchy, reminder tone, and accessibility (text size, contrast).

5. Reflection

The UX process helped identify user needs by shifting from assumptions to real user experiences. It highlighted that chronic condition management is both functional and emotional, leading to design decisions that prioritize flexibility, accessibility, and reassurance.

A key challenge was meeting the needs of diverse users (busy student, elderly user, caregiver). This was addressed by universal design principles: large text, high contrast, minimal steps, and customizable features, then refined through usability testing feedback.
